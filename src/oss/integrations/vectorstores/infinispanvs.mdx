## Setup Infinispan cache

Infinispan is a very flexible key-value store, it can store raw bits as well as complex data type. User has complete freedom in the datagrid configuration, but for simple data type everything is automatically configured by the python layer. We take advantage of this feature so we can focus on our application.

## Prepare the data

In this demo we rely on the default configuration, thus texts, metadatas and vectors in the same cache, but other options are possible: i.e. content can be store somewhere else and vector store could contain only a reference to the actual content.

```
import csvimport gzipimport time# Open the news file and process it as a csvwith gzip.open("bbc_news.csv.gz", "rt", newline="") as csvfile:    spamreader = csv.reader(csvfile, delimiter=",", quotechar='"')    i = 0    texts = []    metas = []    embeds = []    for row in spamreader:        # first and fifth values are joined to form the content        # to be processed        text = row[0] + "." + row[4]        texts.append(text)        # Store text and title as metadata        meta = {"text": row[4], "title": row[0]}        metas.append(meta)        i = i + 1        # Change this to change the number of news you want to load        if i >= 5000:            break
```

# An helper func that prints the result documents

By default InfinispanVS returns the protobuf `Å§ext` field in the `Document.page_content` and all the remaining protobuf fields (except the vector) in the `metadata`. This behaviour is configurable via lambda functions at setup.

```
def print_docs(docs):    for res, i in zip(docs, range(len(docs))):        print("----" + str(i + 1) + "----")        print("TITLE: " + res.metadata["title"])        print(res.page_content)
```
